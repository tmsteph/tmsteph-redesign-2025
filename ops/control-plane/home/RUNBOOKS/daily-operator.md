# Daily Operator

## Goal
Keep the business stack moving without waiting for perfect clarity.

## Default Loop
1. Read `TODAY.md`, `PRIORITIES.md`, and `BUSINESSES.md`.
2. Check whether there is an active production, billing, or customer-facing issue.
3. If yes, fix and verify that first.
4. If no, make one forward move in product, one in revenue, and one in systemization.
5. Write back any important standing changes to the markdown control plane.

## Product Move Examples
- ship a page improvement
- tighten onboarding or billing UX
- fix broken account or checkout flows

## Revenue Move Examples
- improve pricing clarity
- shape a higher-tier offer
- tighten lead capture or checkout friction

## Systemization Move Examples
- create a runbook
- document a decision
- turn repeated work into a repeatable operator step
